
# レポートの提出について

レポート課題では、以下に示す 6 つを提出してください。

1. [設計指針](#1-設計指針)
2. [フローチャート](#2-フローチャート)
3. [ソースコード](#3-ソースコード)
4. [実行結果例](#4-実行結果例)
5. [検証](#5-検証)
6. [考察](#6-考察)

それぞれについて、記載すべき内容や注意事項を以下に示します。

## 1. 設計指針 (何を作るか、どう処理するかを文章で説明する)

設計指針では、プログラムの外部設計と処理の概要を示してください。

### 1.1 外部設計

外部設計はユーザから見えるプログラムの振舞いです。
本実習では特にプログラムへの入力と出力について記述してください。
- **入力** : プログラムの起動時および実行中に、ユーザーがどのようなデータを入力する必要があるのかを、
入力するデータそれぞれについて、その内容や型(整数なのか実習なのか文字なのかなど)などを説明してください。
必要に応じて入力するデータの制約条件(正の数である必要があるや、小文字の半角英数字である必要があるなどの条件)を示してください。

- **出力** : プログラムが出力するデータについて、その内容や型などについて説明してください。
必要に応じて出力データがどのように表示されるのか、そのレイアウトを示してください。

### 1.2 処理の概要

プログラムが、入力されたデータをもとに、どのような手順やアルゴリズムによって出力データを得るのか
その処理の概要を文章で説明してください。必要に応じて数式や図などを用いてください。
ここで説明する処理の概要は、次に示すフローチャートやソースコードと対応するように記述してください。(処理の詳細はフローチャートで示しますので、ここでは概要を説明してください。)

### 注意

- 設計指針はテキストファイルもしくは PDF ファイルで提出してください。
- ファイル名の例：guideline.txt, guideline.pdf


## 2. フローチャート

プログラムの処理の流れを検討し、プログラムの論理的な詳細設計となるフローチャートを作成してください。

### 注意

- フローチャートは 5mm 方眼 A4 サイズのレイアウト用紙に書いてください。
- 指定された課題については、作成したフローチャートに教員または技術職員に確認を受けてサインをもらってください。
- Microsoft Office Lens などのアプリを用いて用紙全体を撮影し、必要に応じて傾きや色の補正を行って読みやすくしたうえで、PDF ファイルに変換して提出してください。
- フローチャートは PDF ファイルで提出してください。
- ファイル名の例: flowchart.pdf


## 3. ソースコード (設計を C 言語で実装する)

作成した C 言語のプログラムを提出してください。
提出する前に、きちんとコンパイルでき、正常に実行できることを確認してください。

### 注意

- ソースコードはテキストファイルで提出してください。ファイル名の拡張子は .c としてください。
- ファイル名の例: exercise.c


## 4. 実行結果例 (実際に動かした結果を示す)

作成したプログラムを実際に実行したときの動作のログを提出してください。

必要に応じて複数の結果例を示してください。
例えば、入力によって分岐があるようなプログラムでは、それぞれの分岐でプログラムが適切に動作をしているかを確認するために、
複数の入力の値に対しての実行結果例を示すとよいでしょう。

### 注意

- 実行結果例はテキストファイル(拡張子 .txt )で提出してください。
- 複数の実行結果がある場合は一つのファイルにまとめてください。
- ファイル名の例 : log.txt


## 5. 検証 (結果が正しいことを根拠をもって説明する)

実行結果例を分析し、作成したプログラムが正しく動作していることを説明してください。
必要に応じて数式や図などを用いてください。

検証では、単に「実行できた」「正しい結果が出た」と書くだけでなく、どのような入力値に対して動作確認を行ったのか、またその入力値を選んだ理由を説明してください。
特に、条件分岐を含むプログラムでは、それぞれの分岐が実行されるような入力値を選んで確認してください。また、繰り返し処理を含むプログラムでは、最小値、境界となる値、通常の値などを用いて確認するとよいでしょう。

実行結果が正しいと判断した根拠も説明してください。必要に応じて、手計算による結果とプログラムの出力を比較してください。

### 注意

- 検証はテキストファイルもしくは PDF ファイルで提出してください。
- ファイル名の例: analysis.txt, analysis.pdf

## 6. 考察 (設計・実装・検証のつながりと改善点を述べる)

考察では、作成したプログラムについて、設計から実装、検証までを振り返り、どのように考えてプログラムを作成したのかを説明してください。

特に、設計指針やフローチャートで考えた内容が、実際のソースコードにどのように反映されているかを述べてください。また、検証で用いた入力値が、問題の条件やプログラムの構造に対して適切であったかを考察してください。

考察には、例えば次のような内容を含めるとよいでしょう。

- 問題文で与えられた条件を、どのように整理したか
- その条件を、どのような分岐処理や繰り返し処理として設計したか
- フローチャートの各処理が、ソースコードのどの部分に対応しているか
- 検証で選んだ入力値が、設計した処理を十分に確認できるものであったか
- 実装してみて難しかった点や間違えやすいと感じた点
- より分かりやすく、読みやすく、間違いにくいプログラムにするための改善点

### 注意

- 考察はテキストファイルもしくは PDF ファイルで提出してください。
- ファイル名例: consideration.txt, consideration.pdf
- 検証と考察を 1 つのファイルでまとめて提出しても構いません。その場合は見出しを付けて「検証」と「考察」の内容が分かるようにしてください。
